# Правила «Коридор» (Quoridor) — зафиксированные для реализации

## Доска и старт

- Доска **9×9**, координаты `(col, row)`, **0-индексация** (обе оси `0..8`).
- **P1** стартует в `(4, 0)`, цель — достичь любой клетки ряда `8`.
- **P2** стартует в `(4, 8)`, цель — достичь любой клетки ряда `0`.
- У каждого игрока по **10 стен** на старте.

## Ход

За один ход игрок делает ровно одно из двух:
1. Передвигает свою фишку.
2. Ставит стену из запаса.

Pass (добровольный пропуск хода) **запрещён**.

## Ходы фишкой

Фишка ходит на одну из **ортогональных** соседних клеток (вверх/вниз/влево/вправо), если:
- клетка в пределах доски,
- ребро к ней не перекрыто стеной.

### Прыжки через соперника (полный стандарт)

Если соседняя клетка **занята соперником**, ребро к ней свободно:

1. **Прямой прыжок**: если клетка позади соперника (в том же направлении) **на доске** и ребро к ней **не перекрыто** — прыгаем туда.
2. **Диагональный прыжок**: если прямой прыжок невозможен (позади стена или край) — игрок может шагнуть на любую **боковую** клетку (перпендикулярно направлению), если:
   - клетка в пределах доски,
   - ребро от соперника до той клетки **не перекрыто** стеной.

Тупика «нет хода» в игре **не бывает** — прыжок всегда выход.

## Стены

### Геометрия

Стена задаётся якорем `(c, r)` ∈ `0..7 × 0..7` и ориентацией `"H"` | `"V"`.
Стена перекрывает **два ребра** между клетками:

- `Wall(c, r, "H")` (горизонтальная) — рёбра `(c,r)↔(c,r+1)` и `(c+1,r)↔(c+1,r+1)`.
- `Wall(c, r, "V")` (вертикальная) — рёбра `(c,r)↔(c+1,r)` и `(c,r+1)↔(c+1,r+1)`.

### Запреты при постановке стены

1. **Дубль**: на том же якоре уже стоит стена той же ориентации.
2. **Крест**: на том же якоре уже стоит стена **противоположной** ориентации (`H`×`V`).
3. **Коллинеарное наложение**: смежный якорь по той же оси уже занят стеной той же ориентации — т.е. сегменты пересекаются.
   - `"H"`: якоря `(c±1, r, "H")`.
   - `"V"`: якоря `(c, r±1, "V")`.
4. **T- и L-стыки разрешены** (перекрёсток касается, но не пересекает другую стену).
5. **Нет стен в запасе**: `walls_left[player] == 0`.
6. **Блокировка пути (BFS)**: после постановки стены у **обоих** игроков должен оставаться путь до своего целевого ряда. Проверяется **BFS**-достижимостью (`has_path`). BFS игнорирует фишки и прыжки — только стены режут рёбра.

## Победа

Игрок побеждает **немедленно**, как только его фишка достигает любой клетки целевого ряда. Ничьих нет.

## Алгоритм проверки пути (BFS)

- Граф: 81 вершина (клетки), рёбра без весов; ребро исключается, если перекрыто стеной.
- Задача — **reachability** (да/нет), не кратчайший путь.
- BFS выбран вместо DFS: попутно даёт расстояние до цели (пригодится для будущего ИИ).
- Запускается при **каждой** попытке поставить стену: гипотетически применили → BFS для P1 → BFS для P2 → если хоть у одного пути нет, ход нелегален.
